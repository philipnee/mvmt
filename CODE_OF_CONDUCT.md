# Code Of Conduct

## Our Pledge

We want mvmt to be a focused, respectful project where people can discuss security, local data access, and developer tooling without hostility.

## Expected Behavior

- Be direct and respectful.
- Critique code, designs, and tradeoffs rather than people.
- Assume contributors are acting in good faith.
- Be careful with security-sensitive information.
- Keep discussions relevant to the project scope.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Discriminatory language or conduct.
- Publishing private information without permission.
- Publicly posting vulnerability details before maintainers have had a chance to respond.
- Repeatedly derailing issues or pull requests after maintainers ask to refocus.

## Enforcement

Maintainers may remove comments, close issues, reject contributions, or block participants who violate this code of conduct.

If you need to report a conduct issue, contact a maintainer privately. For security vulnerabilities, use the process in `SECURITY.md`.
